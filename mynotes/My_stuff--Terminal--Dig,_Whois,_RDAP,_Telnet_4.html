Network Commands

ifconfig - LAN info, see device IPs (local, not public) - this has been replaced by ip addr

ip addr (or ip a for short) - similar to ifconfig but also shows status, can temporarily add/remove IP addresses

ping domain/ip - check connectivity between devices (LAN or public) - specify number of packets with -c (ping -c 5 domain will send 5 packets to the domain)

traceroute domain/ip - shows the route taken to a domain

tracepath domain/ip - same as traceroute but shows latency

mtr domain/ip - traceroute with more advanced features and ping time

dig domain/ip (+noall +answer - this shortens output, +short) - query DNS records (A by default or use ANY for all DNS records)

dig domain/ip +trace - same but follows route

dig domain any @8.8.8.8 - shows all DNS records

dig NS domain +trace - great for checking if subdomain has it's own zone or is in the main domain zone

dig -x ip - reverse dns/ptr lookup (can achieve similar with nslookup ip)

host domain - shows the IP address for a domain (and other DNS records)

whois domain/ip - shows domain/ip info like owners, name servers, dnssec, registrar, etc. just like website

nmap -p 53 ns1.tublat.com (port scanning and testing, can be port 80 for http, 53 for dns, 443 for https) - also gives you rDNS record, port state, and latency

nmap -Pn -p 53 domain/IP (same - scans as if host is open - ignores blocks)

wget - same as curl but uses http, https, or ftp

netstat -s (or -ano) - shows the activity of the various network protocols used by my computer/network

ss - faster alternative to netstat

nmcli dev show | grep 'IP4.DNS' - shows the resolver DNS IP addresses

arp -a - shows the IP addresses that are talking with you computer and the MAC addresses associate with those IP addresses

route - shows where your traffic exits (your computer)

Enter these one line at a time:

nslookup

server 8.8.8.8

set type=TXT

_acme-challenge.tublat.com

(type "exit" and enter to exit)

or all at once:

nslookup -query=TXT tublat.com 8.8.8.8

to measure how long a lookup takes:

time nslookup domain

Check TCP connectivity of IP over a specific port with telnet (are these IPs reachable as web servers?)

telnet <IP Address> 443 or 80 - will either connect or fail

alternatively you can use: curl -v http://IP address (or https for 443)
