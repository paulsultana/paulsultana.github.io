Use Opera First

Then try the following command

Full command

curl -sS -D - -o /dev/null https://tublat.com \

-H "Accept-Encoding: gzip, deflate, br, zstd" \

-H "Cache-Control: no-cache" \

-H "User-Agent: Mozilla/5.0"

1️⃣

curl

curl is a tool for making HTTP requests from the command line.

It lets you manually control things like:

  • headers
  • request method
  • compression
  • redirects
  • HTTP version

So you can simulate exactly what a browser or API client would do.

2️⃣

-sS

-sS

This controls output behavior.

-s

(silent)

Suppresses:

  • progress bar
  • download stats
  • extra output

-S

(show errors)

If something fails, it still prints the error.

So together:

-sS

means:

Run quietly, but still show errors if something breaks.

This keeps output clean.

3️⃣

-D -

-D -

This tells curl:

Print the response headers.

Normally curl prints only the response body.

-D = dump headers.

The - means:

print them to stdout (the terminal)

Example output:

HTTP/2 200

content-type: text/html

content-encoding: br

cache-control: public

This is exactly what you want when debugging compression or caching.

4️⃣

-o /dev/null

-o /dev/null

This tells curl:

Throw away the response body.

/dev/null is a special Linux file that discards anything written to it.

So:

HTML content → discarded

Why?

Because you only care about the headers, not the page itself.

Without this option, curl would dump the entire HTML page into your terminal.

5️⃣

https://tublat.com

This is the URL being requested.

Curl will perform a GET request by default.

So this simulates:

GET / HTTP/2

Host: tublat.com

6️⃣

-H "Accept-Encoding: gzip, deflate, br, zstd"

This adds a custom HTTP header.

Accept-Encoding: gzip, deflate, br, zstd

Meaning:

"I support these compression formats."

These are the common ones:

EncodingMeaning
gziptraditional web compression
deflateolder compression
brBrotli
zstdZstandard

This header is critical for your test because your new WGP logic depends on it.

Your edge will pick one encoding according to priority.

7️⃣

-H "Cache-Control: no-cache"

This sends:

Cache-Control: no-cache

Meaning:

Do not use a cached version — revalidate.

This is useful because otherwise you might hit an old cached object.

For debugging you usually want to force:

edge → origin

so you see the real behavior.

8️⃣

-H "User-Agent: Mozilla/5.0"

This pretends the request is coming from a browser.

Header sent:

User-Agent: Mozilla/5.0

Why this matters:

Some servers behave differently depending on the client:

Examples:

ClientBehavior
curlsometimes disable compression
browserfull optimization

So adding this header helps simulate a real browser request.

What the request actually looks like

The server receives something like:

GET / HTTP/2

Host: tublat.com

Accept-Encoding: gzip, deflate, br, zstd

Cache-Control: no-cache

User-Agent: Mozilla/5.0

What the response might look like

Example output:

HTTP/2 200

content-type: text/html

content-encoding: br

vary: Accept-Encoding

x-location: ac1-UAT

x-protected-by: WEDOS Global Protection

This lets you verify:

✅ compression

✅ cache behavior

✅ which node served the request

✅ WGP headers
